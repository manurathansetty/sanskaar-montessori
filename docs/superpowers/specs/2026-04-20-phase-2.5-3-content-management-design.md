# Phase 2.5 + 3 + Gallery-Categories — Content Management

**Date:** 2026-04-20
**Builds on:** Phase 1 (admin auth) + Phase 2 (Cloudinary images), branch `feat/admin-auth`
**Status:** Draft, pending user approval

## Goal

Give the admin full editorial control over the site's structured content
without touching code. Three feature bundles ship together because they share
the same plumbing (JSON in repo + GitHub-API commits + admin UI patterns):

- **Phase 2.5 — Events config:** add / edit / delete / reorder events on the
  Events page (title, lede, type, meta pills, tags).
- **Phase 3 — Site constants + page-level text:** phone numbers, Google Form
  URL, Maps URLs, school name + tagline, per-page header titles + subtitles,
  per-page CTA banner copy.
- **NEW — Gallery categories:** add / rename / delete / reorder gallery
  sections; deleted categories' photos auto-move to a special "uncategorized"
  bucket so nothing is lost.

## Non-goals (deliberate, deferred)

- Long-form text editing: Founder bios, About page narrative, Home page
  below-the-fold copy stay hardcoded. Long prose is a different UX problem.
- Icon picker for event pills (defaults preserved on migration; new pills
  default to a generic `Tag` icon).
- New founders or new home slots from admin (founders and home are stable;
  dev change if needed).
- In-app drag-reassign UI for moving photos out of `uncategorized` (admin
  can download from `/admin/images/gallery/uncategorized` and re-upload
  via the existing flow).
- Branch-aware commits (all admin saves commit to `master`).
- Multi-admin concurrent edit locking (last-write-wins; acceptable for two
  admins sharing a password).
- Validation library (zod): hand-rolled validation is sufficient for our
  three schemas.
- Long-running background sync of orphaned photos (the rename happens inline
  during category delete; if it fails partway, admin can re-run delete).

### Data flow

**Read (public site):**
- JSON files compiled into the bundle at build time (`resolveJsonModule`)
- Pages import from `site-content.ts` typed wrapper
- No runtime fetch, no /api/load endpoint
- Updates require a redeploy (1-2 min after admin save)

**Write (admin):**
1. Admin edits in form → clicks Save
2. Browser POSTs new content to `/api/save/<resource>`
3. Vercel function:
   - Verifies session (`getSession` from Phase 1)
   - Validates body shape (hand-rolled checks)
   - Reads current file SHA from GitHub via Contents API
   - PUTs updated content + SHA + commit message → commit lands on `master`
4. Vercel auto-detects the commit → preview / production rebuilds
5. Public site shows new content ~1-2 min later

**Delete category (special):**
1. Admin clicks Delete on a category card → confirms in modal
2. POST `/api/categories/delete` with `{ id }`
3. Server iterates Cloudinary `sanskaar/gallery/<id>/*` resources
4. For each: `cloudinary.uploader.rename(oldPublicId, newPublicId)` where new path is `sanskaar/gallery/uncategorized/<original-leaf>`
5. After all renames succeed: commit gallery-categories.json with that entry removed
6. Return `{ ok: true, moved: <count> }`

## JSON schemas

### `gallery-categories.json`

```json
[
  { "id": "classroom",     "label": "The Prepared Environment", "description": "Thoughtfully arranged Montessori classrooms", "order": 1 },
  { "id": "practical-life","label": "Practical Life",           "description": "Everyday skills that build independence",      "order": 2 }
]
```

- `id`: URL slug (lowercase, digits, hyphens). Used as Cloudinary folder name. Auto-derived from label on add (e.g., "Music & Dance" → `music-and-dance`). Validated on save: must match `/^[a-z0-9-]+$/`, no collision with existing.
- `label`: display text. Required, non-empty.
- `description`: subtitle shown under section heading. Optional, defaults to empty.
- `order`: integer. Determines display order on public Gallery page and in admin lists.
- `uncategorized` is a reserved id — implicitly always present; admin cannot add or delete it. It does NOT appear in this JSON file; it lives only as a Cloudinary folder.

### `events.json`

```json
[
  {
    "id": "summer-camp",
    "type": "featured",
    "title": "A Joyful Summer of Learning & Play",
    "lede": "Two weeks of music, yoga, drama, gardening — designed to keep little ones curious, active and happy through the summer break.",
    "imageSlot": "summer-camp",
    "pills": [
      { "label": "When",      "value": "Apr 15 – Apr 30 · Mon–Fri", "icon": "Calendar" },
      { "label": "Time",      "value": "10:00 AM – 1:00 PM",        "icon": "Clock" },
      { "label": "Age Group", "value": "3+ years",                  "icon": "Users" }
    ],
    "tags": ["Best Out of Waste", "Yoga", "Drama & Theatre"],
    "order": 1
  }
]
```

- `id`: URL slug. Auto-derived from title on event create. Validated like category id.
- `type`: enum `"featured" | "programme" | "admissions"`. Maps to badge text + icon:
  - `featured` → "Featured Event" + `Sparkles`
  - `programme` → "Programme" + `BookOpen`
  - `admissions` → "Admissions Open" + `GraduationCap`
- `title`, `lede`: text, required.
- `imageSlot`: references `sanskaar/events/<imageSlot>` in Cloudinary. Defaults to event `id`. Image management stays in the existing image admin (`/admin/images/events/<imageSlot>`) — the event editor links to it, doesn't duplicate it.
- `pills`: array of `{ label, value, icon? }`. Admin can add / remove / edit. `icon` field preserved on migration so existing events keep their varied icons; new pills default to `Tag`. UI does not expose icon selection.
- `tags`: string array. Admin adds via chip input.
- `order`: integer. Determines display order on Events page and in admin list.

## API endpoints

### Common save flow

All save endpoints share this template:

1. `req.method !== 'POST'` → 405
2. `getSession(req, res)` → 401 if `!session.phone`
3. Validate body shape (per-resource hand-rolled checks)
4. If `!process.env.GITHUB_TOKEN`, return 500 `{ error: "github_auth" }`
5. `github.getFile(path)` → returns `{ content, sha }`
6. `github.putFile(path, JSON.stringify(body, null, 2), sha, commitMessage)`
7. Return 200 `{ ok: true }` on success
8. On GitHub 401/403 → 500 `{ error: "github_auth" }`
9. On other failures → 500 `{ error: "github_other", message: <err.message> }`

### `POST /api/save/site`

Body: full `site.json` content.

Validation:
- Required keys present (school, contact, pages)
- All phones have valid `tel` (`/^\+?\d{10,15}$/`) + non-empty `display`
- `primaryPhone` matches some phone's `tel`
- All URLs start with http/https
- All page header / cta titles + subtitles non-empty

Commit message: `chore: edit site config (via admin)`

### `POST /api/save/events`

Body: full events array.

Validation:
- Array (possibly empty)
- Each event: `id` matches slug regex; `type` in enum; `title`, `lede` non-empty; `imageSlot` non-empty slug; `order` integer; `pills` array with `{label, value}` non-empty; `tags` string array
- No duplicate ids

Commit message: `chore: edit events (via admin)`

### `POST /api/save/gallery-categories`

Body: full categories array.

Validation:
- Array (must have at least one entry — site needs a gallery)
- Each: `id` matches slug regex; `label` non-empty; `order` integer
- No duplicate ids; no entry with id `"uncategorized"` (reserved)

Commit message: `chore: edit gallery categories (via admin)`

### `POST /api/categories/delete`

Body: `{ id: string }`.

Flow:
1. Auth check
2. Validate `id` exists in current categories
3. Forbid deletion if `id === "uncategorized"`
4. Read current `gallery-categories.json` from GitHub
5. Use Cloudinary search to enumerate `folder:sanskaar/gallery/<id>/*` resources
6. For each, `cloudinary.uploader.rename(old, new)` where new = `sanskaar/gallery/uncategorized/<original-leaf>`
7. If any rename fails: stop, return 500 `{ error: "cloudinary", message }` (admin can re-run; rename is idempotent — already-renamed photos won't be in the source folder)
8. Once all moved: commit `gallery-categories.json` with the entry removed
9. Return 200 `{ ok: true, moved: <count> }`

Commit message: `chore: delete category "<label>", moved <N> photos to uncategorized (via admin)`

### GitHub helper

`src/lib/github.ts`:

```ts
const OWNER  = 'manurathansetty';      // hardcoded
const REPO   = 'sanskaar-montessori';  // hardcoded
const BRANCH = 'master';               // saves always go to master

function ensureToken(): void {
  if (!process.env.GITHUB_TOKEN) {
    throw new GitHubAuthError('GITHUB_TOKEN env var not set');
  }
}

export class GitHubAuthError extends Error {}

// REST: GET /repos/{owner}/{repo}/contents/{path}?ref={branch}
async function getFile(path): Promise<{ content: string, sha: string }>;

// REST: PUT /repos/{owner}/{repo}/contents/{path}
//   body: { message, content (base64), sha, branch, committer: { name, email } }
async function putFile(path, contentString, sha, commitMessage): Promise<void>;
```

Uses native `fetch` (Node 22's built-in) — no Octokit dependency. Author identity = `Sanskaar Admin <admin@sanskaarmontessori.local>` (commits clearly distinguishable from dev commits in git log).

### `/admin/events` — list view

### `/admin/events/:id` and `/admin/events/new`

One scrollable form:
- **Title** (text input)
- **Type** (dropdown: Featured / Programme / Admissions)
- **Lede** (textarea, ~3 rows)
- **Image** (preview from Cloudinary slot + "**Replace via image admin**" link → opens `/admin/images/events/<imageSlot>` in a new tab)
- **Pills** (dynamic list — each row: `label` input, `value` input, **X** to remove. **+ Add pill** at bottom.)
- **Tags** (chip input: type + Enter to add; click chip's X to remove)
- **Save** / **Cancel** at bottom

For `new`: id auto-generated from title (e.g., "Christmas Concert" → `christmas-concert`). On save, redirects to `/admin/events/<new-id>`. Image upload happens via the link to image admin.

### `/admin/settings` — site.json editor

Three sections (vertical, all visible — no tabs needed for a small form), one **Save All** button at bottom:

**School**
- Name (input)
- Tagline (input)

**Contact**
- Phones — dynamic list. Each row: tel, display, label inputs + **★ primary** radio + **X** delete.
- **+ Add phone** at bottom.
- Registration form URL (input)
- Maps share URL (input)
- Maps embed src (input)

**Pages** — collapsible per-page (Home / Gallery / Events / Founders).
- Each shows the editable fields per the schema (Home: heroBadge + heroDescription; others: header.title + header.subtitle + ctaBanner.title + ctaBanner.subtitle).

### `/admin/categories` — gallery categories

Linked from `/admin/images/gallery` via a "**🗂 Manage Categories**" button at the top of the slot list.

- Cards per category (drag handle, label, description, photo count). Drag-reorder updates `order`.
- Each card: **Edit** (inline → label + description editable) and **Delete** (confirm dialog: "*X photos will move to Uncategorized. This cannot be undone.*")
- Top: **+ Add Category** → inline form (label + description; id auto-slugged from label).
- "Uncategorized" appears as a special read-only entry at the bottom (no Edit / Delete buttons) showing the orphan count.
- Photo count per card requires a Cloudinary count call — fetched once on page mount.

### Toast UI

- Single `<ToastHost />` mounted in `AdminLayout` (so toasts only show on admin pages).
- `useToast()` hook returns `{ toast: { success, error } }`.
- After every save: `toast.success("Saved.")` or `toast.error(msg)` based on response code.
- Toast appears bottom-right, slides in, auto-dismisses after 5s, dismissible via X.

Toast messages by error code:
| code | message |
|---|---|
| `github_auth` | Service connection expired. Contact developer to refresh. |
| `github_other` | Could not save. Please try again or contact developer. |
| `cloudinary` | Could not move photos. Please try again. |
| `validation` | Some fields look invalid. Please check and try again. |
| (network failure / unknown) | Something went wrong. Please try again. |

## Migration

`scripts/migrate-content-to-json.ts` (run once locally):

1. Reads current hardcoded values from `Footer.tsx`, `Events.tsx`, `Founders.tsx`, etc.
2. Writes initial files exactly matching today's site:
   - `src/content/site.json` from Footer phones + EventActions form URL + MapFab URLs + page texts
   - `src/content/events.json` from the 4 events currently in Events.tsx (each with current pills, tags, type derived from badge style)
   - `src/content/gallery-categories.json` from the 8 categories currently in `image-slots.ts` SLOTS.gallery + their descriptions from CATEGORY_META in Gallery.tsx
3. Idempotent — re-running skips files that already exist.

After migration completes successfully, the page-rewrite tasks land. The site looks identical pre/post; only the data source changed.
